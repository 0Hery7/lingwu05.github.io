C程序的生命历程

一个c语言程序从编译到可执行文件的生成经历了下面几个过程：

  1. 预编译

  2. 编译

  3. 汇编

  4. 链接

预编译

通俗而言，预编译就是处理c程序中含有#的语句，例如#include、#define、#if 等。使用命令gcc -E a.c -o a.i 对源文件a.c进行预编译操作，生成预处理文件。

include

#include <a.inc> 或者#include "a.inc"语句会搜索尖括号或者双引号中的文件，并且将其内容直接复制粘贴到源文件中去。对于下面的例子，源代码a.c的输出结果是Hello, world!

其原因是#include语句执行的是文件的复制和粘贴操作。

那么，编译器是在什么地方找到尖括号或者双引号中的文件的呢？又或者编译器在什么地方找？通过查阅官方手册，我们可以使用gcc --verbose a.c来查看编译源文件a.c时编译器具体做了什么处理，这其中也包含了编译器应该在什么地方寻找尖括号或者双引号中的文件。一般而言，编译器会在当前路径去寻找双引号中的文件，而尖括号中的文件的寻找取决于操作系统的一些相关环境变量。

那么我们可以添加想要路径到到尖括号中的寻找路径吗？答案是肯定的。我们可以通过gcc -Ipath（注意-I后面不能有空格，这个是历史遗留问题）将路径path添加到尖括号的寻找路径中。对于命令gcc -I. a.c --verbose，我们可以发现编译器将当前路径也就是.添加到了尖括号中的寻找路径。

编译

编译是将经过预编译的文件转换成汇编文件。可以通过命令gcc -S a.c -o a.s 将源文件a.c转换成汇编文件a.s。或者通过gcc -S a.i将预编译文件a.i转换成汇编文件a.s。

汇编

汇编是将汇编文件转换成二进制的机器语言。可以通过gcc -c a.s 将汇编文件转换成可重定向的二进制文件a.o。或者通过gcc -c a.c将源文件编译并汇编成二进制文件a.o。我们可以通过命令objdump -d a.o对二进制文件或者可执行文件进行反汇编的操作。

链接

链接是将相互依赖的多个二进制文件合并的操作。可以通过gcc a.o b.o -static将二进制文件a.o和b.o合并成可执行的二进制文件。

ISA概述

定义

ISA指Instruction Set Architecture，即指令集体系结构。

具体内容

ISA是一个规约（Specification），它规定了如何使用硬件。

  • 可执行的指令集合，包括指令格式、操作种类以及每种操作对应的操作数的相应规定。
  • 指令可以接收的操作数的类型。
  • 操作数所能存放的寄存器组的结构，包括每个寄存器的名称、编号、长度和用途。
  • 操作数在存储空间存放时按照大端还是小端方式存放。
  • 指令获取操作数的方式，即寻址方式。
  • 指令执行过程的控制方式，包括程序计数器、条件码定义等。

ISA在计算机系统中是必不可少的一个抽象层

  • 没有它，软件无法使用计算机硬件。
  • 没有它，一台计算机不能被称为通用计算机。

ISA和计算机组成的关系

ISA是计算机组成的抽象。不同ISA规定的指令集不同，如，IA-32、MIPS、ARM等。计算机组成必须实现ISA规定的功能，如提供GPR、标志、运算电路等。同一种ISA可以有不同计算机组成，如乘法指令可用ALU或乘法器实现。

位运算

算法移位

算术移位的对象是有符号整数。

  • 左移：高位移出，低位补0。溢出判断：若移出的位不等于符号位，则溢出。

    以8位数为例，1001 1110 << 3 等于 1001 1110。慎用，当移出位不等于符号位会发生溢出。

  • 右移：低位移出，高位补符。可能发生有效数据丢失。

    以8位为例，1001 1011 >> 4 等于1111 1001。慎用，可能会发生有效位丢失。

逻辑移位

逻辑移位的对象是无符号整数。

  • 左移：高位移出，低位补0。溢出判断：高位移出的是1，则左移时发生溢出

    以8位为例，1110 1010 << 2 等于1010 1000。慎用，会发生溢出。

  • 右移：低位移出，高位补0。

    以8位为例，1011 1001 >> 3 等于 0001 0111。随便用。

位拓展

低字节的类型转高字节的类型。

  • 无符号数：0拓展（前面补0）

  • 有符号数：符号拓展（前面补符）

位截断

强制类型转换，直接丢弃高位字节。

计算机系统基础

课程目标

理解计算机是如何生成和运行可执行文件。

学习本课程会收获什么

  • 编程时可以少出错

    • 快速定位程序出错的地方
    • 明白程序是如何在计算机上执行的

  • 编写更好、更快的程序

  • 认识和理解计算机系统

计算机系统的体系结构

计算机系统可以抽象成如下的层次：

  • 应用（软件)
  • 算法
  • 编程
  • 操作系统/虚拟机(如win10、Java虚拟机)
  • 指令集体系结构
  • 微体系结构
  • 功能部件
  • 电路
  • 器件

冯诺依曼结构

基本内容：计算机应由运算器、控制器、存储器、输入设备和输出设备五个基本组件组成。

核心思想：采用存储程序的工作方式。

工作方式：任何要计算机完成的工作都要先被编写成程序，然后将程序和原始数据送入主存并启动执行。一旦程序被启动，计算机应能在不需要操作人员干预下，自动完成逐条取出指令和执行指令的任务。

各基本部件的功能：

    1. 存储器不仅能存放数据，也能存放指令，形式上两者没有区别，但计算机应能区分数据和指令。
    1. 控制器应能自动取出指令来执行。
    1. 运算器应能进行加/减/乘/除四种基本运算，并能进行一些逻辑运算和附加运算。
    1. 操作人员可以通过输入设备、输出设备和主机进行通信。

使用二进制表示数据和指令。每条指令由操作码和地址码两部分组成。操作码指出操作类型，地址码指出操作数的地址。由一串指令组成程序。

使用花括号{}进行变量初始化，我们称这样的初始化为列表初始化。当然，我们也可以使用花括号对变量进行赋值的操作。

在使用花括号赋值时，如果变量的类型与值的类型不相同，如果强制类型转换后发生数据丢失则会编译报错。

默认初始化

如果变量定义时没有指定初值，则变量会被默认初始化，此时变量被赋予了默认值。默认值到底是什么由变量的类型以及定义变量的位置决定。

对于内置类型，它的值由定义的位置决定。定义在任何函数之外的变量（即全局变量）被初始化为0。定义在函数内部的内置类型变量将不被初始化，该变量的值可以是内存中的随机的一个数。

对于类，类的对象如果没有显式地初始化，则其值由类确定。

变量声明和定义的关系

声明使得名字为程序所知，一个文件如果想使用别处定义的名字则必须包含对那个名字的声明。而定义负责创建与名字关联的实体。

变量声明规定了变量的类型和名字，在这一点定义与之相同。但是除此之外，定义还要申请存储空间，也可能为变量赋予一个初值。

变量只能被定义一次，但是可以被多次声明。如果想要在多个文件中使用同一个变量，那么需要将定义和声明分离。此时，变量的定义只能出现在一个文件中，而其他文件用到该变量的文件必须对其进行声明，不可以重复定义。

整型

无符号类型

类型int、short和long都是带符号的，通过在这些类型名前添加unsigned就可以得到无符号类型。

与其他整形不同，字符型被分成了三种：char、signed char和unsigned char。char类型可能是signed char或者unsigned char,具体是哪种由编译器实现。

字面量的类型

整型字面量的类型由它的值和后缀符号决定。十进制字面量值的类型是在能容纳当前值的前提下，int、long和long long中类型最小的那个。八进制和十六进制字面量值的类型可能是带符号型也可能是无符号型。
